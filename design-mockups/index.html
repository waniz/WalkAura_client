WalkAura — Client Design Mockups

2026-05-16 · Targeted redesigns + new quest UI
What this is: static HTML previews of proposed UI changes for the WalkAura Godot client. Open each file directly in a browser. All mockups use shared tokens from _shared.css that mirror scripts/globals/styler.gd.

Priority: Location hub redesign is the focal point — it solves three issues you flagged (poor profession picker, UI overlay on painted bg, no quest entry). Quests page is brand new. Quick-fixes covers smaller wins on inventory, attributes, professions.

What's NOT in these mockups: the actual painted Godot art assets — placeholder Unsplash images stand in for the painted Dragon Lair / minimap. Read structure, hierarchy, and treatment — not literal pixels.

SYSTEM-LEVEL CHANGES (applied to every screen)
Top HUD chrome: Gray placeholder bar → dark-chrome gradient with gold border glow. Avatar + minimap get 2px ornate gold frames with inner shadow. HP/MP bars become painted (gradient fill, inner shadow track, highlight shimmer) instead of flat HTML rectangles. See any mockup for live treatment.

Bottom nav: 5-tab nav with painted icons. Active tab gets gold rim glow + gold tick mark above icon. Quests tab supports notification pip for ready-to-turn-in quests.

Sub-tab pills: Generic blue web pills replaced. Two families: painted tabs (dark chrome, gold underline glow — for inventory/skills/gear) and parchment tabs (cream bg, gold underline — for profile/attributes/professions/achievements).

Tier color discipline: Tier-1 green, Tier-2 blue, Tier-3 purple used consistently across location hero badge, quest cards, rift cards, and item quality tooltips. The painted scene gets a tier-tinted tag so location difficulty reads instantly.
